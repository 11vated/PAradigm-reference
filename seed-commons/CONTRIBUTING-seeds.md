# Contributing to the Seed Commons

The Seed Commons grows three ways: Foundation curation, community pull requests, and the Full-Capacity GSPL Agent's self-bootstrapping loop. This guide is for the first two.

The commons has **zero paywalls, zero rate limits, and no cloud dependency**. Every contribution you make is released under the GSPL Open Specification License and stays forever free.

## Prerequisites

- Read `spec/02-gene-system.md` (the 17 kernel gene types).
- Read `spec/05-sovereignty.md` (canonicalization, SHA-256 over JCS, ECDSA-P256 signing).
- Read `spec/06-gseed-format.md` (`.gseed` binary layout; ADR-009).
- Read `research/088A-canonical-seed-armory.md` for the curation philosophy.

## What a contribution looks like

A contribution is **one or more matched pairs** of files:

```
seed-commons/inventories/<category>/<slug>.gspl         # the GSPL source
seed-commons/inventories/<category>/<slug>.gseed.json   # the grown, hashed, signable payload
```

Optionally:

```
seed-commons/recipes/<subdir>/<slug>.gspl               # if the contribution is a recipe
seed-commons/libraries/<slug>.gspl                      # if the contribution is a library module
```

## Contract checklist

Every `.gspl` source MUST:

1. Declare a `module seed_commons.<category>.<slug>` path matching its location.
2. Import only from `Std.*`, `seed-commons/libraries/*`, or other commons seeds (no external modules).
3. Be **pure** — `fn main() -> <Domain>.Seed pure = seed { … }`. No `Network`, no `Time`, no wall clock.
4. Use only the 17 kernel gene types. Cast any external values with the explicit coercion operators from `language/stdlib.md`.
5. Name every randomness source explicitly (`rng_seed = "<slug>:v1"`), never implicitly.
6. Declare lineage: `parents` MUST be non-empty for derived seeds; primordial seeds list the `libraries/` modules they consume in the `$metadata.consumes` field.
7. Carry a sovereignty block with author thumbprint, royalty_percent (default 0.075), license, and a signature slot.
8. Stay under 500 lines (split larger artifacts into recipes that compose smaller seeds).

Every `.gseed.json` MUST:

1. Validate against `https://paradigm.app/schemas/seed/v1.json`.
2. Have a content hash of the form `sha256:<64 hex>` computed as `sha256(jcs_canonicalize(payload_sans_hash_sovereignty_fitness_metadata_rest))` per `spec/05`.
3. Include a `quality_vector` with all six axes in [0,1].
4. Declare `created_at` in RFC 3339 UTC.
5. Declare `license` explicitly (default `CC-BY-4.0`, may be any OSI/Creative-Commons identifier).
6. For community PRs, the `signature.value` may be a placeholder — the Foundation re-signs on merge.

## How to make a seed revolutionary (raise the quality bar)

The armory's purpose is to be the substrate's first-person voice. Don't contribute forgettable seeds. A revolutionary seed:

- **Grounds in real libraries.** A candle is `libraries/chemistry` + `libraries/physics` + `libraries/materials`, not a hand-waved "warm glow."
- **Carries sovereignty seriously.** Set a royalty_percent (default 7.5%), pick a real license, declare `provenance: human_authored | agent_assisted | fully_agentic`.
- **Exposes at least one cross-domain functor.** A lighting seed that composes into a matching audio ambience via `functor_lighting_to_soundscape` is more valuable than one that renders alone.
- **Uses the quantum gene for stylistic superposition** where appropriate. A seed that collapses at grow-time into one of three cinematographic treatments is richer than a seed locked to one.
- **Declares regulatory networks** for any multi-parameter aesthetic (mood → palette → atmosphere), not hand-tuned constants.
- **Embeds dimensionals** (`Std.Embedding.clip_v1` or equivalent) so the seed is findable in similarity search and usable as a MAP-Elites axis.
- **Declares a resonance profile** for anything that can be heard, tapped, or sung — even if the seed's primary domain is visual.

## The community PR flow

1. Fork the repository.
2. Create your seed files in the correct `inventories/` or `recipes/` subdirectory.
3. Run `validation/grow.ts <path>` locally to grow the seed and regenerate the `.gseed.json` payload hash.
4. Run `validation/determinism.ts <path>` twice and confirm byte-identical output.
5. Run `validation/signature.ts <path>` to confirm structural signability (placeholder signature is accepted).
6. Run `validation/commons-lint.ts <path>` to check the contract checklist above.
7. Open a PR. Include:
   - A 2–3 sentence description of what the seed produces.
   - The reference render hash.
   - Any source-culture attribution (mandatory for culturally-grounded seeds per Brief 086E).
   - The composition graph (auto-generated by `validation/graph.ts`).

## Culture-sensitive seeds

Brief 086E (`research/086E-culture-history-mythology-library.md`) governs culturally-grounded material. If your seed depicts or draws from a living culture:

- Cite your sources in `$metadata.attribution`.
- If possible, list a consultant or community contact in `$metadata.reviewers`.
- Set `$metadata.cultural_sensitivity: reviewed | pending_review | self_authored`.
- Foundation curators may request a source-culture consultant review before merge.

## Agent-authored contributions

If a seed was produced by the Full-Capacity GSPL Agent (`intelligence/gspl-agent-full-capacity.md`):

- Set `provenance: fully_agentic` or `provenance: agent_assisted`.
- Include the agent's session lineage hash in `$metadata.agent_session`.
- Include the validation receipt from the agent's `Validator` sub-agent.
- The determinism, signature, and round-trip tests apply identically — agent provenance does not relax any contract.

## Versioning

- First version is `v1.0.0`.
- Add a seed; bump only for content changes, never for comment-only edits.
- Retain old versions — the commons is **append-only** for signed artifacts. A new version is a new file (`<slug>@v1.1.0.gspl`), not an in-place edit.
- Lineage edges link versions (`operation: "version-bump"`).

## What to do if your seed is rejected

- CI will tell you exactly which contract clause failed.
- Open an issue if the rejection seems wrong — the Foundation treats false positives in `commons-lint` as bugs.
- If you disagree with a curation decision, you can **fork the commons** and federate. Every federation peer is a first-class commons host (Brief 091).

## Why this matters

The Seed Commons is the single largest lever on GSPL's adoption curve. Every canonical seed you contribute:

- Teaches a new user (human or agent) what the substrate can do.
- Becomes training data for the next self-bootstrapping pass.
- Becomes a parent for thousands of breed-derived child seeds.
- Demonstrates to the world that GSPL is not a specification — it is a living, shippable creative frontier.
