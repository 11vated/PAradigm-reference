# 177 — Scene and level editor specification

## Question
What is the creator-facing scene/level editor surface that authors `level.scene` (Brief 172), `ecs.entity` (Brief 153), and `camera.shot` (Brief 155) gseeds with sign-time validation, lineage on every edit, and 2D/3D parity from day one?

## Why it matters (blast radius)
The scene editor is the creator's primary canvas. If it is not signed-by-construction, every save loses the seven-axis discipline at the most-touched surface in the substrate. If it does not match the underlying namespace topology of Brief 172 / 153 / 155, creators learn one mental model in the editor and a different one in the spec — exactly the divergence GSPL exists to prevent. If 2D and 3D are not parity from the schema down (per Brief 156), then 3D becomes a second-class retrofit and v0.4 ships a stitched UX rather than a reach into an already-equipped surface.

## What we know from the spec
- Brief 172 — `level.scene` root with 12 pattern families.
- Brief 173 — tilemap layers and procgen primitives compose into scenes.
- Brief 153 — ECS entity lineage on spawn/destroy with rollback.
- Brief 155 — camera-as-gseed (`camera.shot`, `camera.rig`, `camera.cut`).
- Brief 156 — physics 2D shipped at v0.1, 3D at v0.4.
- Brief 158 — partial save and signed save lineage.
- Brief 057 — modifier-surface DSL — the editor is a modifier surface.
- Brief 052 — lineage-aware time machine — every edit must rewind.

## Findings
1. **Editor as modifier surface.** The scene editor compiles every interactive operation (place, move, rotate, scale, parent, prefab-instance, paint, batch-replace) into a typed mutation against the `level.scene` gseed. Each mutation is signed at commit and lands in the lineage exactly the way a programmatic edit does. There is no "editor private state" outside the gseed. (Brief 057, Brief 052)
2. **Selection model.** Selection is a typed `editor.selection` ephemeral component over entity refs, not a gseed primitive — it never enters the lineage. Selection invariants (must reference live entities, must be cleared on rollback) are enforced by the editor runtime, not by sign time.
3. **Scene graph view.** The editor renders the `level.scene` gseed as a tree with: pattern-family root → composition slots → entities → components. Drag-and-drop reparenting compiles to a `parent_set` mutation. Multi-parent (graph rather than tree) is rejected at sign time per Brief 153's tree invariant.
4. **2D/3D mode toggle.** Editor mode is a viewport setting, not a gseed flag. The same `level.scene` can be viewed in either; the only difference is camera projection and gizmo space. The pattern-family field in `level.scene` declares whether 2D or 3D primitives are usable, and the editor refuses to add 3D primitives in a 2D-declared scene at preview time, before sign time.
5. **Snapping.** Snap settings (grid, vertex, edge, pivot, world, parent-local) are editor-runtime only — never persisted in the gseed. Default grid is 16 units in 2D scenes (matches typical tile sizes from Brief 173) and 0.5 m in 3D scenes (matches Brief 156's physics units). Per-scene snap overrides live in a `level.scene.editor_metadata` typed extension that is signed but excluded from determinism hash.
6. **Prefab and inheritance.** Prefabs are `level.scene` gseeds with a single root entity marked `prefab_root: true`. Instancing creates a typed `prefab_instance` reference, not a deep copy. Overrides are per-component delta records with sign-time validation that the prefab still resolves. This matches Unity's prefab variant semantics but signed and lineage-tracked. (Brief 153)
7. **Gizmos.** Translate / rotate / scale gizmos in 2D and 3D operate on transform components. Each drag emits one mutation per frame at the editor's variable rate but commits a single batched mutation at mouse-up — preventing lineage spam. (Brief 052's time machine traverses commits, not frames.)
8. **Multi-edit.** Editing N entities at once compiles to N parallel mutations under one signed batch. Sign-time validation runs once per batch, not N times. Conflict (e.g., one entity rejects the change) aborts the whole batch atomically.
9. **Undo/redo binding.** Undo is `lineage.rewind(1)`. Redo is `lineage.replay(1)`. These are not editor-private stacks — they share the substrate's lineage history, so undo survives a save/load cycle for free. (Brief 052, Brief 158)
10. **Sign-time errors as gutter markers.** When a mutation would fail validation (e.g., adding a component the entity's archetype rejects, placing a `level.metroidvania_room` chamber that breaks reachability per Brief 172), the editor surfaces the validator's typed error inline at the offending entity with the validator's message and a "fix-up" hint generated by the validator. The error is not silenced — the mutation is rejected and the editor remains in the prior valid state.
11. **Live preview vs sign mode.** Editor has two modes: *preview* (mutations apply to a fork that does not commit until the creator presses sign) and *commit* (every mutation signs immediately). Preview is the default for v0.1 to keep iteration cheap; commit is opt-in for high-stakes scenes. Either way, the gseed's signature only updates on commit boundaries.
12. **Asset palette.** The palette draws from the seed armory (Brief 088A), the chemistry/physics/biology/music libraries (Briefs 081-086H), and federation peers (Brief 100). Drag-from-palette compiles to an `entity.spawn` with the asset gseed as the archetype root.
13. **Pan/zoom/orbit.** Camera controls in the editor viewport are decoupled from `camera.shot` gseeds. The editor has its own ephemeral `editor.viewport_camera` that never enters the scene gseed. A creator can promote the editor camera to a `camera.shot` via a one-click "save as camera shot" button.
14. **Multi-viewport.** v0.1 ships single-viewport. Quad-view (top/front/side/perspective) deferred to v0.2 — adds editor surface area without unblocking any creator workflow at v0.1.

## Risks identified
- **Editor-private state leaking into the gseed.** Easy to accidentally persist viewport camera or selection. Mitigation: sign-time schema rejects any field outside the typed `level.scene` schema; ephemeral state lives in a typed-out namespace the validator excludes by construction.
- **Mutation spam during gizmo drag.** Every-frame mutations would explode lineage. Mitigation: batch on mouse-up, never on mouse-move.
- **Prefab override validation cost.** Resolving an N-deep prefab chain on every edit could be expensive. Mitigation: cache resolved prefab tree per signature root; invalidate only on prefab gseed change.
- **3D in a 2D-declared scene.** Creator confusion if 3D primitives are silently dropped. Mitigation: editor refuses the placement at preview time with a clear error pointing at the scene's pattern-family declaration.
- **Sign-time validator latency.** A heavy `level.metroidvania_room` reachability check on every keystroke would freeze the editor. Mitigation: validator runs only on commit boundary; preview mode shows validator warnings asynchronously after a debounce window.

## Recommendation
Specify the scene editor as a *thin shell* over the `level.scene` gseed namespace: every interaction compiles to a typed mutation, every commit is signed and lineage-tracked, and selection / viewport / snap state lives in a typed-out ephemeral namespace. Ship 2D and 3D parity from the schema down, with viewport mode driven by the scene's declared pattern family. Default to preview mode with debounced validator hints; expose explicit commit mode for creators who want strict per-edit signing. Defer quad-view to v0.2 and live-cooperative editing to v0.3.

## Confidence
**4.5 / 5.** The architecture is mechanical — it falls out of Briefs 153/172/052/057 with very little new design surface. The only novel decisions are the preview-vs-commit mode default and the gizmo batch-on-mouseup policy, both of which have well-known precedents in Unity / Godot / Unreal. Lower than 5 only because debounced validator latency under heavy scenes is empirical and won't be measured until Phase 1.

## Spec impact
- New spec section: **Scene editor surface specification** under the editor namespace.
- Updates `level.scene` schema with a typed `editor_metadata` extension excluded from determinism hash.
- Updates Brief 052 (lineage time machine) with the explicit binding that editor undo/redo IS lineage rewind/replay.
- Adds the preview-vs-commit mode flag as a creator preference.

## New inventions
- **INV-720** — Editor-as-modifier-surface contract: every interactive operation compiles to a typed mutation against the underlying gseed; no editor-private persistent state.
- **INV-721** — Gizmo-batch-on-commit-boundary: drag operations emit one signed mutation per drag, not per frame, to prevent lineage spam while preserving rollback granularity.
- **INV-722** — Preview-mode fork with debounced validator hints: creators iterate in a non-committed fork; full sign-time validation runs only at the commit boundary, with asynchronous warnings during preview.
- **INV-723** — Prefab variant via signed override delta: prefab instances are typed references with per-component delta records, not deep copies, with sign-time resolution validation.
- **INV-724** — Editor-viewport camera as typed-out ephemeral namespace: viewport, snap, and selection state are signed-out by schema, never entering determinism hash or lineage.
